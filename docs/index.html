Workflow

Built like a real machine-learning pipeline

The code is structured so the same workflow can run locally for validation and in Colab for the full data-heavy modeling pass.

01

Extract raw archives

Reads the four DCR zip files and discovers station folders automatically.

02

Normalize workbooks

Handles .xlsx, .xls, and .xlsb files, daily sheets, headers, and timestamps.

03

Build combined dataset

Adds a region label and writes the combined CSV with dataset reports.

04

Train forecasting models

Creates time features, lag features, rolling stats, and pollutant targets.

Explainability

Show why the model made the forecast

The deployable app exposes model feature importance and converts the signal into recruiter-friendly language for pollutant, weather, time, and region factors.

Primary signal PM10 lag features

Recent PM10 values are the strongest available proxy for the next-step PM10 forecast.

Context signal Region: SILTARA

Industrial-zone indicators help the model separate station behavior from city-wide weather patterns.

Weather signal Wind + humidity

Lower wind and higher humidity increase the modeled stagnation risk.

Project Details

            Data preprocessing
            
DCR workbook extraction, timestamp parsing, duplicate handling, and region merge.

          

          

            Feature engineering
            
Lag features, rolling statistics, cyclic time encodings, and station indicators.

          

          

            Modeling
            
Random Forest forecasting for PM2.5, PM10, and SO2 with strategy selection.

          

          

            Deployment
            
Streamlit dashboard, FastAPI endpoint, CLI predictor, Dockerfile, and Render config.
