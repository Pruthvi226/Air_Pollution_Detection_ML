Workflow

Built like a real machine-learning pipeline

The code is structured so the same workflow can run locally for validation and in Colab for the full data-heavy modeling pass.

01

Extract raw archives

Reads the four DCR zip files and discovers station folders automatically.

02

Normalize workbooks

Handles .xlsx, .xls, and .xlsb files, daily sheets, headers, and timestamps.

03

Build combined dataset

Adds a region label and writes the combined CSV with dataset reports.

04

Train forecasting models

Creates time features, lag features, rolling stats, and pollutant targets.

Modeling Evidence

Validated outputs from the current hourly model

These plots summarize the current deployable hourly artifact. PM10 shows the strongest global result, SO2 shows moderate global signal, and PM2.5 requires region-specific modeling because local behavior varies strongly by monitoring region.

Best PM10 R2 0.58
Best SO2 R2 0.43

Model interpretation: The global PM2.5 model has weak performance because PM2.5 behavior varies strongly across regions. Region-specific evaluation improves PM2.5 forecasting significantly, reaching R2 = 0.840 for IGKV and R2 = 0.819 for AIIMS.
